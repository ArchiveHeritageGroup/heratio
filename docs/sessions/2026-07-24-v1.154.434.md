# v1.154.434 (2026-07-24)

v1.154.434: fix(#1425): RiC save crashed writing publication status (bad source_culture column).

Saving on the standalone RiC editor (and the RiC create/edit path) threw 'Unknown column source_culture in field list' on the publication-status write - RicManageController::persist did DB::table('status')->updateOrInsert(..., ['status_id'=>..., 'source_culture'=>...]), but the status table has no source_culture column (object_id, type_id, status_id, id auto_increment, serial_number). Dropped source_culture from that write. The other source_culture writes in persist (event, relation, property) are correct - those tables do have the column. Verified: save sets the status (Draft/Published), re-save updates the single row via updateOrInsert, no exception.
