# v1.154.410 (2026-07-22)

perf(browse): relevance takes the fast path when there is no query. Completes the browse sorts. relevance was excluded from the sidecar because it layers a CASE over the matched text - but that CASE only exists when a query is present. With no query it is nothing more than io.id desc, identical to lastUpdated, and was paying ~17.5s for it. Now routed to the object_id sort key whenever queryFilter is empty, and left on the ordinary path when it is set. One trap handled: relevance pins io.id to DESC regardless of ?dir=, so the sidecar call is forced to desc as well - passing safeSortDir would have silently reversed a page the old path never reversed. Verified: with no query the first ids match the old ORDER BY io.id desc exactly, ascending and descending; with a query the column resolves to null so the CASE path still runs. All eight sorts in both directions smoke-tested, plus relevance with a query
