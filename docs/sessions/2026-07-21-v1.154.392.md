# v1.154.392 (2026-07-21)

fix(home): restore the two 3D digital-twin walkthrough links on heratio.org - HomeController's marketing-host check was never updated for the domain move. v1.154.382 added heratio.org and www.heratio.org to the marketing host list in header.blade.php and ahg-admin-menu.blade.php, but left the identical fallback in HomeController::index recognising only heratio.theahg.co.za. heratio.org therefore rendered the institutional homepage while its navbar behaved as marketing, and because the two exhibition-space.walkthrough links (benson-collection and the-crystal-palace-reconstruction) live in home-marketing.blade.php they simply stopped appearing on the landing page. The marketing hero copy still showed because that is static-page content rendered inside the institutional view, which is why the page looked half right. Host list is now a MARKETING_HOSTS constant with a comment tying it to the two blade files, so the three cannot drift apart again. Verified on dev: with Host heratio.org the marketing view renders with both walkthrough links present and no Browse-by sidebar; with an institutional host the Browse-by sidebar is present and the twin block absent, so customer installs are unaffected. Both walkthrough targets return 200 on prod
