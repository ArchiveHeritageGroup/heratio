# v1.154.424 (2026-07-23)

v1.154.424: feat(#1425): helper line under 'Description standard' + per-sector Level filter on the RiC/DACS/RAD forms.

(1) Added a helper line under the 'Description standard' selector on the archival create + generic-edit forms explaining that switching the standard reshapes the field set below (ISAD(G) / RiC-O / DACS / RAD / MODS / Dublin Core); the edit form also notes matching values already captured are kept.

(2) The base archival form already scoped the Level-of-description dropdown to the sector (via level_of_description_sector -> the nine 'archive' levels), but the RiC/DACS/RAD standard editors built their level dropdown from the full taxonomy 34 - so switching to one of those standards on the archival form reverted the dropdown to all 27 levels across every sector. RicManageController / DacsManageController / RadManageController now apply the same level_of_description_sector filter (sector='archive', ordered by display_order, guarded by Schema::hasTable with a full-list fallback for a minimal install). MODS/DC have no Level selector so are unaffected. Verified each now returns the nine archival levels.
