# v1.154.402 (2026-07-22)

perf(browse): indexable alphabetical sort via a title-sort sidecar. Completes the browse work: v1.154.401 fixed text search, leaving the no-query landing page at ~18s, of which the main result query was ~10s and almost all of that was ORDER BY i18n.title. Root cause is the same one behind the term-page slowness in v1.154.399 - information_object_i18n.title is varchar(1024) and its only covering index stores a 191-char PREFIX, and MySQL will not use a prefix index to satisfy an ORDER BY at any length, so every alphabetical browse filesorted all 454,393 rows. Widening the index cannot help (prefix indexes are unusable for ordering, and utf8mb4 caps a key at 3072 bytes / 768 chars against a 1024-char column) and the base AtoM table is off-limits, so the sort key is projected into a sidecar per ADR 0001: information_object_title_sort (object_id, culture, title_sort) with a FULL-length index on (culture, title_sort, object_id). Keyed by culture because the stored value is the already-resolved COALESCE(NULLIF(title,''), source-culture title) that browse actually displays. Adds TitleSortService, ahg:display-rebuild-title-sort, idempotent first-boot table creation, and an hourly withoutOverlapping rebuild - the projection is one set-based statement (~3s for 454k rows) so the sidecar self-heals rather than requiring all ~27 code paths that write titles to know it exists; refreshFor() is there for callers that want an immediate update. Browse falls back to the old ORDER BY whenever the sidecar is missing or unpopulated, so a fresh install is slow-but-correct, never wrong. COLLATION BUG CAUGHT IN TESTING: collation decides ordering and differs per install - dev is utf8mb4_0900_ai_ci, atom is utf8mb4_general_ci - and a hardcoded general_ci sidecar ordered punctuation the opposite way from the base column on dev. alignCollation() now detects and matches the source column's collation on create and on every rebuild. Verified after the fix: identical ordering to the old path, asc and desc, over 500 rows. Mirrored into database/core/00_core_schema.sql so CI and fresh installs carry the table
