# v1.154.387 (2026-07-21)

fix(ric): #1418 - RiC Explorer overview showed 3 unlabelled Rule nodes instead of the catalogue. Root cause was a wrong setting, not code: ahg_settings.fuseki.fuseki_endpoint pointed at the openric-model dataset (the RiC-O ontology itself - 0 RecordSets, 0 Records, top predicates changeNote/isDefinedBy/label) instead of the ric dataset holding the actual records. Settings corrected separately. These code changes make the failure diagnosable instead of silent: (1) getData's DB fallback tested empty(nodes), which three orphan Mandate/Rule/Mechanism entities satisfied, so buildOverviewGraphFromDatabase never ran even though the graph contained nothing a user would recognise - replaced with hasRecordBearingNodes(), which looks for RecordSet/Record/Person/CorporateBody-class nodes; (2) new mergeGraphData() keeps triplestore context entities alongside the database graph, de-duplicating by URI, rather than discarding one or the other; (3) extractLabel() only understood slash-separated URIs with numeric local ids, so URN-style (urn:ahg:ric:rule:912155) and non-numeric (.../rule/text_900328) identifiers fell through to the literal string Unknown - added patterns for both, giving 'Rule 912155' and 'Rule text_900328'. Verified on dev: overview went from 3 nodes / 0 edges / 3 Unknown labels to 177 nodes / 198 edges / 0 Unknown, with the real entity mix (44 RecordSet, 21 Record, 35 Instantiation, 13 Person, 9 CorporateBody, 7 Place, 1 Family). Remaining and unrelated: the dashboard reports 'Orphaned Triples: 0' while orphan rules are present, and the sync log shows repeated batch failures
