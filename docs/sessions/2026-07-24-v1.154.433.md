# v1.154.433 (2026-07-24)

v1.154.433: fix(#1425): expand the RiC form sections by default so all fields are visible.

On the standalone RiC editor (/admin/ric-manage/edit/{slug}) and the RiC swap on the create/edit form, all 40 fields rendered but only the Identity accordion was open - the other seven sections (Content and structure, Conditions, Related materials, Instantiations, Events, Access points, Description control) were collapsed, so it looked like the RiC edit fields were missing until you clicked each header. Set every section to open by default (accordion-collapse collapse show + un-collapsed buttons); the headers still collapse/expand on click. No field or persist change - purely the default expanded state. _fields compiles.
