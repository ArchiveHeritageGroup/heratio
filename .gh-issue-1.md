Title: Add model_manifest JSON column to inference and model config

Description:

As part of the AI Governance & Sovereignty plan (docs/ai-governance-issue.md), we need to capture structured model provenance with each inference and store model manifests at the config level.

Proposal:
- Add a JSON/JSONB column `model_manifest` to `ahg_ai_inference` (and optionally `ahg_llm_config`) to store a canonical manifest for the model used at inference time.
- Schema keys (suggested): model_id, model_name, model_version, checkpoint_uri, artifact_hash, training_dataset_ids, fine_tune_info, tokenizer_id, tokenizer_hash, declared_capabilities, attestation {report_uri, hash}, publisher.
- Migration: write a DB migration that adds the JSONB column (nullable), and backfills `model_manifest` for recent inference rows from current LlmService dispatch logs where available.
- Tests: add unit tests validating storage and retrieval, and an integration test that an inference record includes the manifest after LlmService::generateSuggestion.

Files referenced:
- docs/ai-governance-issue.md
- packages/ahg-provenance-ai/src/Services/InferenceService.php
- packages/ahg-ai-services/src/Services/LlmService.php

Acceptance criteria:
- Migration adds `model_manifest` column and deploys cleanly.
- Inference records created after the change include a non-null `model_manifest` when model metadata is available.
- Code paths do not break existing consumers.
