# v1.154.247 (2026-07-05)

fix(onboarding): #1399 repair ahg:display-auto-detect + add ahg:onboard-atom-db one-shot. The classify command was broken — it queried/wrote a phantom information_object.display_type column (SQLSTATE 42S22), so records were never classified and the standard GLAM browse showed almost nothing over an AtoM DB. Rewritten to delegate to the real DisplayTypeDetector (display-standard -> level -> parent -> events -> media, defaulting to archive) writing the correct display_object_config table, plus a fast set-based --bulk-archive path for homogeneous archives / large corpora. New ahg:onboard-atom-db runs the full derived-layer build in the correct order for a fresh Heratio-over-AtoM database: migrate -> build-closure -> es-reindex (parallel + bulk-mode) -> display-auto-detect (archive default) -> display-reindex, with --dry-run/--workers/--fresh-index/--classify-archive/--skip-migrate; idempotent and re-runnable. Verified on dev: classify no longer errors, detects mixed types and defaults unmatched to archive, idempotent; onboard pipeline registers and lists in order.
