# v1.154.268 (2026-07-10)

fix(#1357): ahg-dacs-manage language-field collision — the Conditions 'Language & script notes' (languageNotes = language/script of the material) and the Description-control 'Language(s) of description' (DACS 8.3) both POSTed name=languageNotes, so one silently overwrote the other. Split the Description-control field to a distinct languageOfDescription property (generic property/property_i18n table, new name only — no schema change); both fields now persist independently.
