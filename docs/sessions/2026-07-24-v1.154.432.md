# v1.154.432 (2026-07-24)

v1.154.432: fix(#1425): complete the rico-property labels across the RiC form and record page.

The RiC-O form and record page labelled most fields with their RiC-O property (rico:title, rico:history, ...) but were inconsistent - the label was missing on some fields entirely.

- RiC form (ric-manage _fields): the Description control area had no rico labels while every other area did. Added them: Description identifier (rico:identifier), Rules or conventions (rico:descriptiveNote), Sources (rico:hasSource). Publication status is deliberately left unlabelled - it is a Heratio publication state, not a RiC-O property.
- RiC record page (show-ric): added the missing Appraisal (rico:descriptiveNote) and Accruals (rico:descriptiveNote) rows to Content and structure (they were absent, so they never displayed even when filled), and a new Description control section (Description identifier / Rules / Sources) with the same rico labels. Empty fields stay hidden, as before.

Convention kept: standard ISAD fields that serialize to RiC-O carry their rico hint; RiC-O-only concepts already had theirs. Both views compile.
