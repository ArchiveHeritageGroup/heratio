# v1.154.415 (2026-07-23)

feat(ric): in-editor RiC-O conformance validation (#1425 A3). Wires the engine's ShaclValidationService into the RiC editor: RicManageController::validateRic() runs validateBeforeSave() (SHACL shapes + mandatory RiC-O elements + referential-integrity checks) on the serialized record and passes the result to edit.blade, which shows a non-blocking conformance panel - green 'Conforms to RiC-O' when clean, a red list of violations (e.g. "Mandatory field 'rico:identifier' is missing for Record") and an expandable advisory-warnings section otherwise. Runs on every GET, which is also where a save redirects, so it doubles as the post-save conformance report without blocking the save. Guarded by class_exists(ShaclValidationService) so it no-ops on a minimal install. Verified end to end: a complete record shows the green conforms panel; blanking the identifier surfaces the mandatory-field violation in an alert-danger list, restored after
