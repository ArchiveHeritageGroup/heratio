# v1.154.397 (2026-07-22)

fix(term): cap the term-show sidebar tree so wide taxonomies stop exhausting memory. Every term page under a wide parent died with 'Allowed memory size of 536870912 bytes exhausted (tried to allocate 148914176 bytes)' - on atom.theahg.co.za term 110 (root of taxonomy 30) has 558,715 direct children, and both the siblings and narrower-terms queries were unbounded ->get() with a correlated child_count subquery per row, so a single page hydrated half a million rows. The view only ever drew the first 50, so the rest were loaded purely to be discarded. Adds TermController::TREE_MAX (50) as a hard LIMIT on both queries, reuses the existing narrowerCount rather than re-counting, and passes narrowerTotal/siblingsTotal so the '... and N more' affordances report true totals instead of the capped ones. Also caps the previously-unbounded NT list in the Hierarchical terms field, and adds the missing sibling-side 'and N more at this level' line. The List tab (paginated, 25/page) remains the way to see everything. Verified on dev against an 800-wide group: HTTP 200, tree capped at 50, count reads 'and 748 more at this level'
