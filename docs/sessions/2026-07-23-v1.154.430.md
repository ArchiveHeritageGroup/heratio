# v1.154.430 (2026-07-23)

v1.154.430: feat(#1425): add 'Add child' to the RiC record page (parented, RiC-default).

RiC-O records render show-ric.blade, which had only an 'Edit (RiC-O)' button and no way to add children - so a RiC record was a dead end for building a hierarchy. Added an 'Add RiC child' split-button next to Edit (create-gated via AclService): the main button creates a RiC child (the parent is RiC, so RiC is the sensible default); the caret opens 'Add child description in' -> RiC-O / ISAD(G) / DACS / RAD / MODS / Dublin Core. Each uses the ?standard=<code>&parent=<id> deep-link, so the new description opens in that standard, its field set swapped in, and parented to the RiC record. Verified: show-ric compiles; main link -> ?parent=<id>&standard=ric (preselects RiC + sets parent). Complements the archival record's split-button (v1.154.428).
