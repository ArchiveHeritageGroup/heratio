# v1.154.277 (2026-07-11)

fix(#1404): audit-trail never mis-writes user_id — AuditLogger resolves user_id ONLY from the staff 'web' guard, so a non-staff / second-guard principal (public claimant, API consumer, service account) can no longer leak into ahg_audit_log.user_id (FK'd to user(id)). A non-staff actor's identity is still captured in the no-FK username/user_email columns. Fixes both the silent FK-reject drop and the cross-actor misattribution.
