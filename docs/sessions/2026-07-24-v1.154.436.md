# v1.154.436 (2026-07-24)

v1.154.436: change(#1425): 'Add new' creates a child in the parent's own standard (drop the standards dropdown).

Replaced the per-standard 'Add new' split-button (v1.154.428) on the record action bar with a plain 'Add new' that creates the child in THIS record's description standard - children inherit the parent's standard instead of always defaulting to ISAD or making the user pick. _actions-bar resolves the record's display_standard_id to its taxonomy-70 code and appends ?standard=<code> to the create link (parent_id set); a plain ISAD record (no display standard) gets the plain create path. Applies on both the ISAD and RiC record pages (both use _actions-bar). Verified: RiC record -> Add new = ?parent_id=NNN&standard=ric; ISAD record -> ?parent_id=NNN; compiles.
