# Decision: Differential privacy - considered and rejected

**Decision (2026-06-30): Heratio does NOT implement differential privacy, by design. Use small-cell suppression (hide aggregate counts below a threshold, e.g. n < 5) for any statistical/dashboard release instead.**

**Reason:** Differential privacy protects *aggregate statistical releases over large populations* by deliberately adding calibrated noise. Heratio's purpose is the opposite - holding and presenting *individual, authentic* records - so DP is incoherent on records and directly conflicts with the content-authenticity chain (fixity / PREMIS / C2PA prove data is unchanged; DP changes it). The only genuine re-identification exposure is small cells in aggregate dashboards, which small-cell suppression solves more cheaply, understandably, and without ε-budget accounting or utility loss. POPIA/GDPR do not require DP, and no GLAM buyer requests it. Revisit only if Heratio pivots to publishing population-scale statistical products or ML training data over sensitive personal records. Privacy today is handled by `ahg-privacy` (PII scan, redaction, DPIA, Article 30, DSAR) + ODRL access policies + audit trail.
