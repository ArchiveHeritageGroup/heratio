# v1.154.416 (2026-07-23)

fix(security): digital-object viewer must not hand the master original to anonymous viewers. Confirmed gap: the 'Open in new window', 'Open full size', Fullscreen and inline controls in the shared digital-object viewer (used by archival + DAM + museum show pages) pointed a NOT-logged-in visitor straight at the master original - $masterUrl for open-in-new-tab, and $imgSrc = $masterUrl ?: $refUrl for the image viewer/open-full-size/fullscreen/OSD deep-zoom - with no auth gate; verified an image record served 2 anonymous master links before this fix. Now: public (not authenticated) viewers get the derivative only (reference 141, else thumbnail 142) for all those controls, and when NO derivative exists the Open/Fullscreen controls are HIDDEN rather than falling back to the master. $imgSrc resolves to the derivative for the public (also fixes the OpenSeadragon source and the inline image link). The PDF branch is gated too: a public viewer gets a web-optimized reference PDF if one exists, otherwise the thumbnail preview with a 'log in to view the full document' note - never the master PDF inline. Authenticated viewers are unchanged (still get the master); the existing redaction reroute and the already-@auth-gated 'Download file' control are preserved and compose correctly. Verified anon vs authed across an image record (2 master links -> 0), a no-derivative record (controls hidden, master withheld) and a PDF. SCOPE: this fixes the viewer CONTROLS per the agreed 'controls only' decision; the master file is still directly fetchable at its static /uploads path by anyone who has the URL (nginx serves it) - gating the file itself (auth_request / streaming route) is a separate defense-in-depth follow-up. 3D-model and native video/audio still use the master (out of the image/PDF scope of this rule)
