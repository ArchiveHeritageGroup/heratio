# v1.154.427 (2026-07-23)

v1.154.427: feat(#1425): deep-link standard preselection on the create/edit form + fire the field swap on load.

Takes the one genuine advantage of the PSIS/AtoM add form (arriving already set to a description standard via URL) and adds it to Heratio's form - which keeps its superior TRUE per-standard field sets (RiC/DACS/RAD/MODS/DC each with their own field model, persist and serializer, plus the structured instantiation/event editors) rather than AtoM's single relabelled ISAD schema.

- create() accepts ?standard=<code> (preferred - a PORTABLE taxonomy-70 natural key like 'ric', which survives across databases) and ?standardId=<term id> / ?display_standard_id=<term id> (per-DB id, kept for PSIS-twin / cross-system link compatibility), resolving either to the taxonomy-70 term and preselecting the 'Description standard' dropdown. Also accepts ?parent= as an alias for ?parent_id= (PSIS parity).
- The create + generic-edit swap JS now fires on LOAD, not only on change: if the driver's selected option is non-ISAD (a deep-link, or a record's own standard on edit, or a non-ISAD choice restored after a validation error) it fetches and swaps in that field set immediately. This also fixes a latent bug where a failed save reverted the form to ISAD fields regardless of the chosen standard.

Result: one form, two ways in - deep-linkable per-standard entry points / 'add child in standard X' / context actions via ?standard=ric&parent=123, plus the in-form dropdown for switching. Verified ?standard=ric, ?standardId=NNN and ?standard=dacs&parent=1 each preselect the right standard; both blades compile.
