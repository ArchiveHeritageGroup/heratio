# v1.154.412 (2026-07-22)

fix(release): stop git add -A sweeping unrelated files into releases. bin/release staged the entire working tree, so a selective git add before invoking it was meaningless: v1.154.400 committed composer.json + composer.lock registering a gitignored client-only package under a message that said 'no new dependencies', producing a tag whose composer.json cannot resolve in any clone, and v1.154.401 swept the reverse back out under a browse-performance message. Adds RELEASE_EXCLUDE (default composer.json composer.lock; opt back in with --include-drift or RELEASE_EXCLUDE=). Moves staging ahead of the session-log generation so its 'Files in this release' list reports what is actually committed rather than what happened to be pre-staged - v1.154.400 logged 1 file while committing 5, and an accurate list is the cheapest guard against this whole class of mistake. Adds a meaningful-content guard: because version.json is rewritten before staging, a naive empty-index check can never fire, so holding back drift would otherwise cut a content-free release; the guard now ignores version.json and rolls the version bump back on abort so a refused release cannot leak a stale bump into the next one. Verified in an isolated repo across all three paths: drift held back, drift-only aborts with rollback, --include-drift commits deliberately.
