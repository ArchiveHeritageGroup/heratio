# v1.154.428 (2026-07-23)

v1.154.428: feat(#1425): 'Add child in a standard' split-button on the record page (Create RiC child).

The record action bar's 'Add new' button becomes a split button: the main button still creates an ISAD child under the record; the caret opens a menu to create a child pre-set to a description standard - RiC-O, ISAD(G), DACS, RAD, MODS or Dublin Core - via the v1.154.427 deep-link (?standard=<code>&parent=<id>, portable natural-key codes). 'RiC-O (Records in Contexts)' is listed first per the request.

Clicking a standard opens the create form already on that standard's field set (swap fires on load) and parented to the current record. Verified: the RiC-child link -> HTTP 200, driver preselected to 'ric', parent_id=<record>; actions-bar compiles; child-link URLs are ?parent=<id>(&standard=<code>). Scope: the archival record action bar (_actions-bar.blade.php), included by show.blade line 1435.
