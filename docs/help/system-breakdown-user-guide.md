# System Breakdown User Guide

## Overview

The System Breakdown is a four-level capability tree at **/help/system-breakdown** that maps everything the platform can do, from the whole system down to individual features. Level one is Heratio itself; level two is the entities it manages (records, actors, repositories, and so on); level three is the aspects of those entities (Provenance, Condition, PII, AI, and more); level four is the concrete features within each aspect. You drill in level by level, follow a breadcrumb back out, search by keyword, and use a minimap to keep your place in a large tree. Open it at **/help/system-breakdown**.

---

## What it does

The System Breakdown turns the platform's full capability set into a single navigable map:

- It presents a **four-level tree**: Heratio (level 1) -> entities (level 2) -> aspects such as Provenance, Condition, PII, and AI (level 3) -> features (level 4).
- It lets you **drill down** through those four levels to reach any feature, and **breadcrumb** back up to any ancestor.
- It provides a **search** so you can jump straight to a capability by keyword instead of clicking through each level.
- It shows a **minimap** that gives an at-a-glance view of where you are within a large tree and lets you move quickly to another branch.
- It serves as an orientation aid for new users, evaluators, and administrators who want to understand the breadth of the platform in one place.

The aim is discoverability: a single, structured answer to "what can this system actually do, and where does each capability live?"

---

## How to use it

1. Open **/help/system-breakdown** (for example `https://your-site.example/help/system-breakdown`).
2. Start at **level 1 (Heratio)** and select an **entity** to move to level 2.
3. Choose an **aspect** of that entity, such as Provenance, Condition, PII, or AI, to move to level 3.
4. Select a **feature** at level 4 to see the specific capability within that aspect.
5. Use the **breadcrumb** at the top to step back to any earlier level without losing your place.
6. Use **search** to find a capability by keyword and jump directly to it, skipping the manual drill-down.
7. Use the **minimap** to see the whole tree at a glance and navigate to another branch in one move.

---

## Good to know

- The four levels read as a sentence: this system, manages these entities, which have these aspects, which offer these features - drilling down narrows the scope at each step.
- The breadcrumb always reflects your current path, so you can confirm exactly which entity and aspect a feature belongs to before acting on it.
- Search is the fastest route when you already know the name of a capability; the drill-down is better when you are exploring what exists.
- The minimap is most useful in large trees where scrolling through every branch would be slow.
- The breakdown is a map of capabilities, not a permissions list - seeing a feature here does not mean every user can access it, since access rules still apply in the live areas of the platform.
