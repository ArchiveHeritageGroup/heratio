# v1.154.426 (2026-07-23)

v1.154.426: feat(#1425): structured RiC Event editor (repeatable rico:Event rows) - stream B2.

Completes the structured RiC editors (B1 instantiations shipped v1.154.425). Adds a repeatable Event editor to the RiC-O create/edit form so an operator records datable happenings in a description's life - custody transfer, publication, accumulation, reproduction - as first-class rico:Event / AtoM event rows instead of free-text.

- UI: an 'Events' accordion in the RiC _fields partial - repeatable rows (event type from taxonomy 40 EXCLUDING Creation, display date, agent/actor label, ISO start/end, note), reusing the [data-repeat-*] document-level delegation from B1.
- Persist: RicManageController::syncEvents diffs submitted rows against the record's existing NON-creator events (type != 111; Creation stays owned by the creators/date widgets) and creates/updates/deletes them as object(QubitEvent)+event+event_i18n (agent + note in i18n). fieldsPartial/edit load existing non-creator events.
- Serializer: no change needed - RicSerializationService::getDatesForRecord already emits every event as rico:hasDateRangeSet -> rico:hasDateRange, so new events flow through all RiC-O surfaces.

Verified: create 2 (Custody + Publication) -> object+event+event_i18n rows, creators (type 111) untouched; update+delete diffs correctly; serializer emits the DateRange; all blades compile. #1425 structured editors (instantiations + events) now complete.
