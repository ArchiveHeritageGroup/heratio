# v1.154.368 (2026-07-17)

feat(c2pa): #1387 - in-PDF (embedded) Content Credentials, not just sidecar. New C2paService::embedInPdf() writes the signed manifest INTO a PDF as a C2PA-associated embedded file (/AFRelationship /C2PA_Manifest) via an append-only incremental update - the original bytes are untouched (proven byte-identical prefix) and the manifest is a standard PDF attachment, never a page-content object, so the document renders identically in every viewer (verified: pdfinfo reads it, pdfdetach extracts c2pa.json). Hard-guarded to classic-xref, unsigned, unencrypted, simple-catalog PDFs; every output is round-tripped (extractC2paFromPdf) before acceptance, else it degrades to the existing sidecar. Wired into WriteC2paSidecar listener, ProvenanceRecordService, and VerifyObjectDownloadController download path. ahg-c2pa tests 14/14
