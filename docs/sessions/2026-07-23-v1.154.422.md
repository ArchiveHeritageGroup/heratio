# v1.154.422 (2026-07-23)

v1.154.422: feat(#1425): persistent PER-RECORD RiC view lens - mechanism + actor (proof slice).

Replaces the session-global ric_view_mode toggle (which bled the RiC lens onto every record for the whole session) with a durable, per-record choice keyed on object.id, so each record remembers whether it displays in the RiC relational lens or its flat description standard - independently of other records and of the session.

- New sidecar table ric_entity_view (object_id PK, view_mode), self-installed idempotently in AhgRicServiceProvider::boot() (same guarded CREATE-IF-NOT-EXISTS pattern as ric_deprecated_entity), so it needs no migrate step and degrades gracefully on a minimal install.
- New AhgRic\Services\RicViewModeService: mode()/isRic()/set(), all guarded by a cached Schema::hasTable probe, defaulting to config('ric.default_view','heratio') when a record has no stored preference.
- RicController::setViewMode persists per-record when an object_id is POSTed, falling back to the old session key when none (so any un-migrated caller still works).
- _view-switch partial takes an objectId: active-state reads, and the toggle writes, the persisted per-record mode via a hidden object_id input.
- Wired the ACTOR show page as the proof slice: passes objectId to the switch and gates the RiC lens + panels on RicViewModeService::isRic($actor->id).

Verified: table self-installs on boot; set(a1,ric) leaves a2 at heratio (no bleed); controller POST persists per-record and returns the effective mode; actor show.blade compiles and references the per-record service with no leftover session key. Remaining 13 entity types roll out next by the same pattern.
