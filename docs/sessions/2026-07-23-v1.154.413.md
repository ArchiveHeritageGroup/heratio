# v1.154.413 (2026-07-23)

feat(ric): RiC-O as a selectable description standard - ahg-ric-manage plugin + Phase A1 (#1425). Makes Records in Contexts (RiC-O 1.0) a first-class description standard alongside ISAD(G)/DACS/RAD/MODS, cloned from the ahg-dacs-manage pattern and wired through the AtoM standards mechanism. NEW self-contained package packages/ahg-ric-manage: RicManageController (source_standard='RiC-O 1.0'; display-standard dropdown reads taxonomy 70, NOT the stale 52 the DACS clone still carries), edit.blade with a RiC-O JSON-LD preview, config seam ric-manage.edit_middleware (default acl:update; OpenRiC host overrides to token auth) so one package serves two hosts, and RicTemplateTermSeeder that idempotently self-seeds the taxonomy-70 'ric' term on boot so the standard appears in the create dropdown with no manual step. Standalone-install guarantee (Johan): the package hard-depends only on ahg/ric (which depends only on ahg/core); every reach into the engine or a sibling package is guarded (class_exists/View::exists/Route::has), so it installs on a bare client server and degrades cleanly. CORE INTEGRATION (additive, guarded): SettingHelper gains resolveObjectTemplateView() - per-RECORD template resolution keyed on the record's own display_standard_id -> taxonomy-70 code -> show-{code}, which lights up show-ric AND incidentally fixes DACS/RAD/MODS (previously the template was chosen only by a global setting, so switching one record's standard did nothing); standardCode() + standardEditRoute() map a record to its per-standard editor, falling back to the generic editor when the record sets no standard or the *-manage package is absent. InformationObjectController::show() now resolves per-record and passes engine-serialized ; edit() hands a RiC/DACS/RAD/MODS/DC record off to its own editor via a guarded redirect. show-ric.blade.php (under io-manage so the resolver finds it) renders the record in RiC-O element framing + a JSON-LD panel + the ahg-ric entities panel. Route lookahead gains ric-manage; visible-elements gains a ric_headings block. Verified end to end on dev: term auto-seeds (code=ric), route registers (ahgricmanage.edit), a record set to RiC renders show-ric (RiC-O badge + JSON-LD), /informationobject/{slug}/edit 302-redirects to the RiC editor, the form loads with the JSON-LD preview, save stamps source_standard='RiC-O 1.0' and persists; a normal record is unaffected and standardEditRoute falls back to the generic editor for standards without a package; ahg-ric engine suite 83/83. A2 (contextual entities + export/OAI surfaces) and A3 (superset depth + in-editor SHACL) follow as their own releases behind the same guards
