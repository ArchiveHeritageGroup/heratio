# v1.154.438 (2026-07-24)

v1.154.438: feat(#1425): RiC-O download button + embed the RiC Explorer/entity viewer on the record page; fix stale RiC-panel guards.

(1) The RiC record page (show-ric) dumped the full RiC-O JSON-LD as inline text. Replaced it with a download card - JSON-LD / Turtle / RDF-XML buttons - streaming from the metadata-export ric route (same serializer). Added a JSON-LD branch to MetadataExportController::downloadRic (application/ld+json attachment) and allowed 'jsonld' on the ric.{ext} route (was ttl/rdf only).

(2) Embedded the RiC Explorer graph (_ric-panel, Cytoscape 2D/3D) and the tabular entity viewer (_ric-entities-panel) on the RiC-O record page, so it shows the same explorer PSIS's /test-ric does, for this record.

(3) Fixed a stale guard both panels carried: they gated on the session-global session('ric_view_mode'), which the per-record RicViewModeService (v1.154.423) replaced - so they had stopped rendering. Now they honour the per-record mode plus an alwaysShow flag (the RiC-O page passes it, since that page IS the RiC view). This ALSO repairs the base ISAD show page's RiC graph/entity panels in per-record RiC mode.

Verified: ric.jsonld -> HTTP 200 application/ld+json; show-ric renders the graph panel, the entity table and the three download buttons; all compile.
