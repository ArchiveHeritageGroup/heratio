# v1.154.418 (2026-07-23)

feat(io): dynamically swap the field set when the description standard changes (create + generic edit). Picking a standard in the new 'Description standard' driver (moved above the accordion so it persists) now AJAX-swaps the whole field accordion to that standard's own field set, and Save persists via that standard's controller - no page reload. Mechanism (reuse, not rebuild): each standard's field accordion is extracted into a reusable _fields partial (nullable $io -> empty on create, populated on edit) that the standard's own editor now @includes; the controller gains fieldsPartial() (render the partial for create/edit) + persist() (the save logic, extracted from edit()'s POST branch). InformationObjectController::standardFields() (GET /informationobject/standard-fields?code=&slug=) dispatches by taxonomy-70 code to the owning *-manage controller, falling back to the built-in _isad-fields partial for isad / unknown / not-yet-wired standards; store()/update() call dispatchStandardPersist() to overlay the chosen standard's fields (no-op for ISAD). All guarded by class_exists/method_exists so a minimal install or an unwired standard degrades to ISAD. STAGE 1 wires ISAD + RiC (RicManageController got fieldsPartial()+persist(), edit.blade refactored to @include _fields); DACS/RAD/MODS/DC fall back to ISAD until wired in stage 2. Also: the display-standard dropdown carries term.code (data-code) as the swap driver, and edit()'s displayStandards now passes the full term collection (was plucked to {id:name}, which also fixes the edit dropdown). Verified end to end: create->pick RiC->RiC fields load->save creates with source_standard='RiC-O 1.0'; edit an ISAD record->switch to RiC->fields swap->save flips source_standard ISAD->RiC-O 1.0; switch back to ISAD restores cached fields; unwired DACS falls back to ISAD; plain ISAD create unchanged; standalone RiC/DACS/RAD/MODS editors still render; 2487 views compile
