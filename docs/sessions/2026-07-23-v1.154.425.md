# v1.154.425 (2026-07-23)

v1.154.425: feat(#1425): structured RiC Instantiation editor (repeatable rico:Instantiation rows) - stream B1.

Adds a repeatable, structured Instantiation editor to the RiC-O create/edit form, so an operator records a description's manifestations (original holding, microfilm, digital surrogate) as first-class rico:Instantiation entities instead of free-text notes. Auto-derived instantiations (from digital objects) are untouched and coexist.

- Schema: ric_instantiation gains a 'source' VARCHAR ('auto'|'manual'), self-installed idempotently in AhgRicServiceProvider::boot(); the one-time digital-object backfill stays 'auto', the editor writes 'manual'.
- UI: an 'Instantiations' accordion in the RiC _fields partial - repeatable rows (label/title, carrier type, MIME/medium, extent value+unit, location/note) via a <template> + a generic [data-repeat-*] document-level delegation added to create.blade / edit.blade / the standalone ric-manage editor (delegation survives the standard-swap innerHTML replacement).
- Persist: RicManageController::syncManualInstantiations diffs submitted rows against the record's existing manual instantiations (create/update/delete) via RicEntityService (full RiC entities: object+slug+i18n+Fuseki), never touching 'auto' rows. fieldsPartial/edit load existing manual rows for editing.
- Serializer: RicSerializationService::getInstantiationsForRecord now also emits the manual rico:Instantiation rows (identifier/carrier/extent/mime/note), so they flow through every RiC-O surface (OAI rico / GraphQL ricO / metadata-export).

Verified: create 2 -> 2 manual rows (object+slug+i18n, source='manual'); update+delete diffs correctly; serializer emits 3 (1 auto + 2 manual) with carrier/extent; auto rows never clobbered; all blades compile. Event editor (stream B2) next.
