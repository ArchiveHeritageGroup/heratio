# v1.154.380 (2026-07-21)

fix(library): #1412 - remove the duplicate FRBR work-key schema (standardise on library_item, per the #1412-1414 decision). The interop-backbone migration had scaffolded a parallel, never-wired pair - library_item.frbr_work_key (varchar 64) + table library_item_frbr_override (target_work_key) - with zero readers/writers repo-wide, so any consumer of them saw empty values. Migration drops both; the live pair (library_item.work_key + library_work_override + frbr_override_type, populated by ahg-biblio-frbr WorkKeyService) is kept. Verified: dead dropped, live intact, ahg-library 198/198
