# v1.154.396 (2026-07-22)

i18n(ci): fix the hardcoded UI strings so the Translation lint gate goes green. The 'Hardcoded UI string scan' has been red since v1.154.380 (61 candidates against a cap of 50) - strings never wrapped in __(), so the multilingual layer can never see them. Wraps 40 genuine user-facing strings across 14 views: the /compare/atom marketing table (16 headings + th), the HTR field-service overlay and index selects, migration-assessment options, and singles across c2pa, cart, digital-object upload, provenance, annotations, rights and repository upload-limit. Count 61 -> 22, and TOTAL_THRESHOLD ratchets 50 -> 30 to lock the gain in. The remaining 22 are deliberate and must NOT be wrapped - TK/BC Local Contexts codes, MIME types and PRONOM ids, URL and path examples, the literal word RESTORE a user types to confirm a restore, and two linter false positives - so 22 is the floor, not a backlog. Wrapping created 40 new keys, which pushed the 'Locale key extraction' gate from 64 to 117 missing, over its cap of 100; ran ahg:translation-extract to harvest them into lang/en.json (18,702 -> 18,819 entries, purely additive: 0 altered, 0 removed). Both gates now pass; 2,487 views compile
