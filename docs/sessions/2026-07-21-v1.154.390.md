# v1.154.390 (2026-07-21)

fix(archaeology): #1422 - place seeded terms in the nested set and closure tree. The v1.154.389 seeder inserted terms with parent_id but no lft/rgt and no term_closure entry, leaving all 162 invisible to both structures. That matters because term.lft still drives ordering in places (DamService, genre access points) and #1333 added term_closure alongside it as a dual-write, so a term with neither sorts unpredictably and is missed by closure-based ancestor queries. Terms are now inserted without lft/rgt and placed in one contiguous block afterwards, shifting the tree once rather than once per term, with closure maintained through the shared ClosureMaintenanceService rather than this module encoding the table shape. New --repair-tree option backfills terms already seeded, for instances that ran .389; it is idempotent and safe to re-run. Verified on dev: root 110 expanded 1470 to 1794 (exactly 2x162), 162/162 terms carry lft, 162 appear in term_closure, zero overlapping lft values and nothing outside the root range. Note: ahg:build-closure --table=term --verify reports one node short of a self-row, which is the pre-existing term 'Preservation Master' (Digital Object Usages, id 1254842, null lft) created by another process - unrelated to archaeology and left untouched
