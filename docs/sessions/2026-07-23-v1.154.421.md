# v1.154.421 (2026-07-23)

v1.154.421: feat(#1425): GraphQL RiC-O interchange + grouped 'Administration area' on the archival-description form.

(1) ahg-graphql - new ricO(id: Int!) query returns a record's RiC-O (Records in Contexts) JSON-LD via the shared ahg/ric RicSerializationService (the third interchange surface after the OAI 'rico' metadataPrefix and the RiC-O metadata-export download). Guarded by class_exists(RicSerializationService) so it degrades on a minimal install, and gated by the SAME published + cultural-protocol (TermProtocolGate) access control the archive resolvers use, so draft/restricted records return 'not found' and are never disclosed; documented in getSchemaInfo. Endpoint stays admin-only.

(2) create.blade + edit.blade now group the standard-agnostic controls - Description standard (the swap driver), Publication status, Source language - into one persistent 'Administration area' that lives OUTSIDE the swappable #standard-fields accordion, so it survives a standard swap. Previously on edit the admin area was INSIDE the swap target and vanished when the standard changed; on create there was no admin-area grouping at all. Publication is de-duplicated by hiding each *-manage _fields partial's own publication block when rendered through the /informationobject/standard-fields swap route (ric/dacs/rad/mods/dc); the standalone standard editors still show it. Fixes the missing/duplicated Administration area reported on informationobject/add.
