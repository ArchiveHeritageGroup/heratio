# v1.154.384 (2026-07-21)

fix(library): #1413 #1414 - standardise Z39.50 import and BIBFRAME export on library_item; remove the phantom library_biblio_* scaffold. #1413: importMarc was NOT dead code - Z3950Controller::import/importBatch call it, so the Z39.50 Import buttons were actively broken; it now creates real library_item records via ahg-library CopyCataloguingService, and the four library_biblio_* writers are gone. Also fixed on that path: parseMarcRecord read leader pos 10 (indicator length, always 2) as the directory length so its loop never ran and it returned an empty array for EVERY binary record - now delegates to ahg-library and handles MARCXML, which is what yaz actually returns for USmarc; result.blade.php referenced an undefined service variable (fatal on the result browser you must pass through to reach Import); batch import counted failures as successes. #1414: new BiblioWorkRepository projects the catalogue onto BIBFRAME (Work = library_item work_key cluster, Instance = library_item, Item = library_copy, Agent = library_item_creator deduped with MARC relator roles); BibframeService, BibframeSerialisationService, BibframeController and GraphEditorController all read through it and RDF import creates real records via LibraryService. Also fixed: importRdf returned the OpenRiC proxy response instead of its stats so the import screen always reported 0 works/0 instances; agent.blade called toDateString() on a query-builder string (fatal once non-empty); .jsonld 500'd on missing ml/json-ld and now 501s with an actionable message; title falls back across cultures so a record catalogued only in af no longer exports as Untitled. Bonus: My Tasks open-task count unified across the dashboard tile, header badge and My Tasks page via SpectrumWorkflowService::countOpenTasks - each procedure ends on its own state name so the hardcoded terminal-state lists over-counted, and myTasks excluded final states as a union across ALL procedures which could hide live work. Verified: dashboard reads 16 works/19 instances (was permanently 0), RDF round-trips and re-import is idempotent, both MARC syntaxes parse; ahg-z3950 100/100, ahg-library 198/198. Outstanding: ahg-biblio-frbr reads the same phantom scaffold and can now reuse BiblioWorkRepository
