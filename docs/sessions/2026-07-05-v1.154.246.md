# v1.154.246 (2026-07-05)

perf(search): #1398 make ahg:es-reindex NARSSA-scale — kill the per-document N+1 + parallelise. New HierarchyQueryService::batchAncestorIds() batches the ancestor lookup for a whole chunk in one query (closure fast-path + single nested-set fallback), replacing the per-row ancestorIds() call that dominated a full reindex; verified 0 mismatches vs the old path and ~39x faster on the ancestor step. Command gains --workers (shard the informationobject reindex across N subprocesses by id-range), --id-range, --bulk-mode (refresh_interval=-1 + replicas=0 during load, then restore + refresh), --shards, --no-progress; small indices (actor/term/repository) stay inline. Verified on dev: 3-worker parallel run shards+completes, ES ancestors match DB exactly. Baseline to beat: AtoM 8M in 3 days — build the closure (ahg:build-closure) for the indexed fast path at that scale.
