# v1.154.386 (2026-07-21)

feat(i18n): #1420 - add ahg:translation-extract and harvest 13,955 missing keys into lang/en.json. Heratio had no extraction step, so UI written with correct __() wrapping never reached the locale files; Laravel's __() falls back to rendering the key, so those strings displayed as English with nothing reporting a problem. en.json covered only 20% of the 17,483 __() keys in the codebase. New command walks packages/app/resources/routes for __()/trans()/@lang() with literal arguments, skipping namespaced (package::key), dotted PHP-array refs and variable arguments, and fills en.json using the key as its own value. Behaviour-neutral: __() already returned the key when absent, so nothing renders differently - the strings simply become visible to translation tooling. en.json 4,747 -> 18,702 entries, 0 pre-existing values altered or lost, written via temp file + rename so a failure cannot truncate the locale file, using the existing ksort + JSON_PRETTY_PRINT convention. Refuses to auto-fill any locale but en, since elsewhere the key is English masquerading as a translation and would inflate coverage. Reports but never deletes the 1,219 now-unused keys, as a static scan cannot see runtime-built keys. --fail-above=0 --dry-run is CI-ready. Corrected baseline: af covers 18.8% of the real UI, zu 4.6%, ve/ts 1.5% - previous figures measured against en.json, which was itself a fifth of the interface
