# v1.154.395 (2026-07-22)

fix(i18n,csp,taxonomy): de-brand remaining user-visible Spectrum strings; fix the d3 CSP gap (#1422); chunk the SKOS export whereIn (#1424). (1) Spectrum sweep: 13 distinct displayed strings across 15 blades plus 4 artisan command descriptions now use the house term 'Museum Procedures' already established by the earlier pass - Filter by museum procedure, Museum Procedures Dashboard, Museum procedure compliance dashboard, Museum procedure chain rules, My Museum Procedure Tasks, Museum Procedure History Export, Enable Museum Procedures, Museum procedure settings, Museum CSV, Procedure:, and CCO cataloguing standards. Zero __() strings and zero command descriptions still contain the trademark. Internal code deliberately untouched: 60 /spectrum routes and 109 Spectrum class references intact, per the rule that only displayed text changes. (2) #1422: five views loaded d3 from https://d3js.org, which is not in HeratioCspPreset's script-src allowlist, so d3 was CSP-blocked on every installation - reported by an external adopter on a clean install. Repointed to cdn.jsdelivr.net/npm/d3@7, the origin master.blade.php already documents using for exactly this reason, rather than adding a fifth CDN origin to the policy. Note the tags are NOT redundant despite the layout also loading d3: master loads it after the footer include while these views run d3 inside @section('content'), so deleting them throws 'd3 is not defined'. (3) #1424: TermController::exportSkos passed a whole-taxonomy id list to whereIn; atom.theahg.co.za's Places taxonomy has 196,322 terms against MySQL's 65,535 placeholder cap, so /term/export/skos/rdf-xml?taxonomy=42 failed outright with error 1390 - a hard failure, not a slow query, for any vocabulary over ~65k terms. Chunked at 5,000 across the other_name and note lookups plus CrossMatchService::forTerms, which fixes every caller of that service. Verified against the live 196,322-id set: unchunked reproduces error 1390, chunked completes in 40 queries
