# v1.154.302 (2026-07-13)

fix(chatbot): suppress ungrounded assistant answers instead of displaying them - GuardrailService::checkGrounding already flags a reply whose claims are not supported by the retrieved sources, but the result was only logged while the answer (with fabricated records/citations) was still shown. Now when grounded=false the reply is replaced with 'I could not find anything in the catalogue matching your query' and the invented citations are dropped. Also harden the system prompt: never infer specific titles/authors/holdings from a vague description (a 'science fiction' record must not become 'Star Wars by George Lucas')
