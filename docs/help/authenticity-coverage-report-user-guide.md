# Authenticity Coverage Report User Guide

## Overview

The Authenticity Coverage Report is an admin dashboard at **/admin/c2pa/coverage** that shows how much of the collection carries content credentials. Instead of asking "is this one item signed?", it answers "how complete is our authenticity coverage overall?" - and it breaks that figure down **by repository**, so you can see which collections are well covered and which still have gaps. It gives administrators a single view of authenticity coverage across the whole platform. Open it at **/admin/c2pa/coverage**.

---

## What it does

This dashboard measures and reports content-credential coverage across the collection:

- It provides an **admin coverage view** at **/admin/c2pa/coverage** summarising how much of the collection carries content credentials.
- It reports coverage **by repository**, so each collection's level of authenticity coverage is visible side by side.
- It surfaces **gaps** - the parts of the collection that do not yet carry content credentials - so they can be prioritised.
- It turns per-item authenticity into a **collection-wide measure**, useful for assurance, reporting, and planning.
- It gives administrators evidence of how thoroughly content credentials have been applied.

The aim is oversight: a clear, repository-level picture of authenticity coverage and where it is still missing.

---

## How to use it

1. Open **/admin/c2pa/coverage** (for example `https://your-site.example/admin/c2pa/coverage`).
2. Read the **overall coverage** figure to see how much of the collection carries content credentials.
3. Review the **breakdown by repository** to compare coverage across collections.
4. Identify repositories with **low coverage** and plan to apply content credentials where they are missing.
5. Revisit the dashboard over time to confirm that coverage is improving as gaps are closed.

---

## Good to know

- Coverage is a measure of how widely content credentials have been applied, not a judgement of any single item's authenticity - use the verification surfaces to check an individual record or file.
- The repository breakdown is the most actionable part of the report: it tells you where to direct effort to raise overall coverage.
- A rising coverage figure over time is a good signal that authenticity practices are being applied consistently across the collection.
- This is an administration dashboard, so access follows the platform's admin access rules.
- Coverage reflects what currently carries content credentials, so newly credentialed items will be included the next time the figures are calculated.
