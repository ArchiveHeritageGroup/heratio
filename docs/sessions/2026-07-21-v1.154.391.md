# v1.154.391 (2026-07-21)

ci(i18n): gate __() keys missing from lang/en.json, so the #1420 extraction cannot silently reopen. Adds a second job to the Translation lint workflow running ahg:translation-extract --dry-run --fail-above, alongside the existing hardcoded-string scan. The two checks guard the two distinct ways a string fails to reach a translator: never wrapped in __() at all (existing lint), or correctly wrapped but never harvested into en.json (new). The second is the more dangerous because __() falls back to rendering the key, so the string displays as English and nothing reports a problem - that is how en.json came to cover only 27% of the interface before #1420. --dry-run means CI never writes the locale file. EXTRACT_THRESHOLD is set to 100 rather than 0, matching this workflow's existing documented convention that the point is not to hit zero but to stop new code pushing the count up; the count stands at 64, all from ahg-archaeology, and the threshold should ratchet down as that clears. Failure output names the remedy (run ahg:translation-extract, commit en.json) because 'key exists in code but not in the locale file' is not an obvious diagnosis. Verified by simulating the job locally: passes at the 100 cap with 64 missing, and correctly exits non-zero when the cap is 0
