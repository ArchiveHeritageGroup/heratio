# v1.154.423 (2026-07-23)

v1.154.423: feat(#1425): roll persistent PER-RECORD RiC view lens to all remaining entities + composite-key redesign.

Completes the per-record RiC lens (mechanism + actor shipped v1.154.422): every entity show page now reads/writes a durable per-record view preference instead of the session-global toggle.

REDESIGN - re-keyed the ric_entity_view sidecar from a bare object.id to a COMPOSITE (entity_type, entity_id). Not every wired entity is an AtoM object-subtype: 'loan' is a standalone custom table whose small auto-increment ids would collide with object ids in a shared single-column key. The composite key namespaces every type. AhgRicServiceProvider::boot() drops the empty v1.154.422 single-column table and recreates the composite one (idempotent, self-installing). RicViewModeService now takes (entityType, entityId); the controller reads entity_type + object_id; _view-switch takes an entityType.

Wired 18 show pages, each passing its entityType + record id to _view-switch and gating the RiC lens on RicViewModeService::isRic(type, id): actor / repository (ISDIAH) / function (ISDF) / accession / donor / rights-holder / term (SKOS) / storage / loan; sector shows dam / gallery / library / museum; the archival IO show.blade + the display-standard variants show-ric / show-dacs / show-rad / show-mods.

Verified: table migrates to composite; actor#5=ric and loan#5=heratio stay INDEPENDENT (no cross-type collision - the point of the redesign); unset records default to config('ric.default_view','heratio'); controller round-trip persists (entity_type,object_id); all 19 blades compile; no session('ric_view_mode') left anywhere.
